DES-CBC Encryption

DES (Data Encryption Standard) is included for educational comparison only. Its 56-bit key was broken in 1998 and it is not secure for modern use. This demo shows how it works alongside AES.

⚠️ Security note: DES uses a 56-bit effective key. The EFF DES Cracker broke it in 22 hours in 1998. Never use DES in production — use AES-256 instead.

Security Q&A (Report Answers)

Which algorithm did you choose, and why?
AES-256 is the primary algorithm. It is the NIST-standard symmetric cipher with a 256-bit key space that is computationally infeasible to brute force.

What is the role of IV?
The IV (Initialisation Vector) is a random 16-byte value that ensures two identical plaintexts produce different ciphertexts. Without it, patterns in the plaintext leak into the ciphertext (ECB mode weakness).

How was decryption verified?
SHA-256 hashes of the original and decrypted bytes are compared. A match proves not a single byte was altered.

What is one security limitation?
The AES key is stored in a metadata file on disk. In production, keys must be stored in an HSM or KMS service. The hybrid mode partially mitigates this by RSA-wrapping the AES key.
